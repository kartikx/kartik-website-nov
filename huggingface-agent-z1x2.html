Multi-Agent Website Cloner

Multi-Agent systems fascinate me. I am particularly interested in leveraging their collaborative potential to model complex tasks. To explore this, I developed a multi-agent system to clone websites. I used the open-source multi-agent framework AutoGen to build this.
The system comprises of 3 agents:
  1. Coder: Writes the code to clone a website.
  2. Evaluator: Evaluates the coder's attempts and provides feedback.
  3. Screenshotter: Takes screenshots of the target and cloned website.
Since multi-agent systems allow fine-grained LLMs that can interact with each other iteratively, they are a lot more accurate at solving complex problems than single agent systems. However, this benefit comes at the cost of increase code complexity with each added agent, as well as a non-determinism due to the nature of LLMs.
There is a clear need for a stable, open-source, computer-use framework that can allow everyone to build and deploy multi-agent systems. I would love to contribute towards this goal at Hugging Face.

Model Sharding for LLM Pre-Training

I am also motivated by the urgency to democratize training large ML models. LLMs promise revolutionary capabilities, however only a handful of big tech players can train them today. I am inspired by the current work that Hugging Face is doing in this space.
To explore this area, I evaluated several distributed training strategies, such as Data Parallelism, Tensor Parallelism and Pipeline Parallelism using DeepSpeed to train GPT-2 on heterogenous GPUs.
Based on these experiments, our team found that optimizing for Data Parallelism is the most efficient. We used this observation to develop an algorithm to search effective model sharding strategies on heterogenous device clusters, reducing search time by 15% compared to the state-of-the-art.

Distributed Systems experience

My biggest strength is my systems engineering skills. Prior to joining University of Illinois, I was a software engineer in the Azure Networking team at Microsoft.
I developed a distributed network troubleshooting service to process gigabytes of daily telemetry across 10,000+ routers, reducing incident resolution time by 30%. To do this at scale, I architected an event-driven microservices pipeline using serverless functions and messaging queues. I also designed an interactive React web application for collaborating incident troubleshooting and telemetry visualization.
More recently, I have been developing distributed systems in Go. I developed a fault-tolerant distributed file system supporting file operations with high availability and eventual consistency. To maintain fault tolerance, I developed a membership protocol, which detects node failures under 5 seconds and ensures 3x replication at all times for all files.
